# Architecture — Environment Variables

The single reference for every setting the stack understands. There is exactly **one source of truth** for variable names and defaults — this table, [`.env.example`](../../.env.example), and [`compose/docker-compose.yml`](../../compose/docker-compose.yml) are kept in lockstep; the reconciliation pass catches drift.

Configuration varies by environment; code does not. Every value here comes from the environment, never baked into an image. For the security upgrade path see [security/secrets-management.md](../security/secrets-management.md) and [ADR-0005](../adr/0005-environment-configuration.md).

## Project & edge

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `COMPOSE_PROJECT_NAME` | `infra-lab` | compose | prefixes all container/network/volume names |
| `PUBLIC_DOMAIN` | `:80` | Caddy | `:80` = serve HTTP on localhost (dev). Set a real hostname to enable automatic TLS |
| `ACME_EMAIL` | `ops@example.com` | Caddy | required for ACME when `PUBLIC_DOMAIN` is a hostname; ignored for `:80` |

## PostgreSQL

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `POSTGRES_USER` | `infra` | postgres, backend, exporter | superuser at volume-initialization time |
| `POSTGRES_PASSWORD` | _(empty) → bootstrap)_ | postgres, backend, exporter | **secret**; `make bootstrap` fills it |
| `POSTGRES_DB` | `appdb` | postgres, backend, exporter | created at first init |
| `POSTGRES_PORT` | `5432` | compose `ports` | host-side publish dev only; removed by prod overlay |

## Redis

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `REDIS_PASSWORD` | _(empty → bootstrap)_ | redis, backend, exporter | **secret**; sets `requirepass`; empty = no auth |
| `REDIS_PORT` | `6379` | internal only | never published to the host (dev or prod) |

## Backend

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `BACKEND_PORT` | `8080` | backend | internal listen port; not published |
| `LOG_LEVEL` | `info` (`debug` in dev, `warn` in prod) | backend | pino log level |
| `COMMIT_SHA` | `dev` (prod sets `IMAGE_TAG`) | backend | surfaces in `/api/status` and `/metrics` |
| `SHUTDOWN_TIMEOUT_MS` | `10000` (prod `15000`) | backend | hard cap for graceful drain on `SIGTERM` |

> `POSTGRES_HOST`/`POSTGRES_PORT`/`REDIS_HOST`/`REDIS_PORT` are passed to the backend by compose with **internal** values (`postgres`, `5432`, `redis`, `6379`), not the `.env` host-port fields. See the `x-backend-env` anchor in [`docker-compose.yml`](../../compose/docker-compose.yml).

## Observability

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `PROM_RETENTION` | `15d` | prometheus (CLI flag) | honored via `--storage.tsdb.retention.time` in compose |
| `GRAFANA_ADMIN_USER` | `admin` | grafana | — |
| `GRAFANA_ADMIN_PASSWORD` | _(empty → bootstrap)_ | grafana | **secret**; `make bootstrap` fills it |

> The scrape interval is a **literal** (`15s`) in [`monitoring/prometheus.yml`](../../monitoring/prometheus.yml) — Prometheus does not interpolate env vars in its config file. Only `PROM_RETENTION` is env-driven (compose CLI flag).

## Backups

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `BACKUP_RETENTION_DAYS` | `7` | `scripts/backup.sh` | prune `pg_dump` files older than N days |

## Production overlay only (`docker-compose.prod.yml`)

| Variable | Default | Used by | Notes |
| --- | --- | --- | --- |
| `REGISTRY` | `ghcr.io/your-org` | prod overlay | image registry; images pulled from `${REGISTRY}/infra-lab-*` |
| `IMAGE_TAG` | `latest` | prod overlay | the pinned tag prod pulls; set to a `v#` tag by release |
| `BACKEND_REPLICAS` | `2` | prod overlay `deploy.replicas` | honored with `--compatibility` |
| `BACKEND_MEMORY_LIMIT` | `512m` | prod overlay | per-replica cap |
| `BACKEND_CPU_LIMIT` | `0.50` | prod overlay | per-replica cap (cores) |

## Secrets handling

`POSTGRES_PASSWORD`, `REDIS_PASSWORD`, and `GRAFANA_ADMIN_PASSWORD` are the three secrets.

- In dev they live in `.env` (gitignored) and are generated by `make bootstrap`.
- In prod, prefer SOPS or a secrets manager. `.env` is a starting point, not the destination — see [security/secrets-management.md](../security/secrets-management.md).
- Secrets are **never** baked into images. `Dockerfile`s use `COPY` for code only; `.dockerignore` rejects `.env`, `*.key`, `*.pem`.
